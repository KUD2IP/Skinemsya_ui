# skinemsya_ui

Фронтенд проекта **«Скинемся»** — Telegram Mini App для совместного сбора и разделения расходов между участниками. Тёмный интерфейс с фирменными зелёными акцентами, плавные анимации, скелетоны на всех экранах и уникальный визуальный почерк, не похожий на типовую «ИИ-генерацию».

> Бэкенд проекта живёт в отдельном репозитории: `../skinemsya_java`. **На фронте реализуется только тот функционал, который реально есть на бэке.** Список доступных эндпоинтов — в [docs/API.md](docs/API.md).

---

## Что это за проект

«Скинемся» (от рус. *скинемся* — «давайте сложимся деньгами») помогает группе людей вести общие траты: скидываться на события, делить чеки, отслеживать кто кому должен и удобно закрывать долги. Приложение запускается внутри Telegram как Mini App.

**Текущая стадия:** ранний MVP. На бэкенде готовы авторизация через Telegram и профиль пользователя. Остальные домены (группы, события, чеки, долги, платежи) запланированы, но API ещё нет — поэтому на фронте они **не реализуются**, пока не появятся соответствующие эндпоинты.

---

## Стек в двух словах

- **React 18 + Vite + TypeScript** — скорость разработки и сборки.
- **vanilla-extract** — типобезопасная дизайн-система в виде токенов (без Tailwind).
- **Ark UI** — доступные headless-примитивы, поверх которых построены собственные компоненты.
- **Motion** — плавные анимации и переходы.
- **Phosphor Icons (duotone)** — фирменный набор иконок.
- **Rive** — анимированный экран загрузки.
- **@telegram-apps/sdk-react** — интеграция с Telegram Mini Apps.
- **TanStack Query / Router**, **Zustand**, **react-hook-form + zod**, **ky** — данные, роутинг, состояние, формы, HTTP.

Подробнее и с обоснованием выбора — в [docs/TECH_STACK.md](docs/TECH_STACK.md).

## Главные правила проекта

1. **Никакого функционала, которого нет на бэке.** Источник истины — `../skinemsya_java` и [docs/API.md](docs/API.md).
2. **Никакого Tailwind и дефолтных палитр** — это выдаёт ИИ-генерацию. Только токены дизайн-системы.
3. **Все переиспользуемые компоненты** живут в `src/shared/ui` отдельными модулями.
4. **Каждый экран** имеет скелетон загрузки; запуск приложения сопровождается анимацией Rive.
5. **Полная адаптивность** под все разрешения и safe-area Telegram.
